# `lines_256` Dataset Note

## Purpose

`lines_256` is the repo-local name for the N=256 lines/structured NPZ pair used for PyTorch architecture experiments.

Use this dataset when you want a single fixed `N=256` lines benchmark without switching to the dual-profile canonical promotion flow.

## Dataset Identity

- Dataset alias: `lines_256`
- Runbook profile name: `custom_npz_pair_n256`
- Train NPZ: `outputs/hybrid_resnet_structural_rerun_20260226T110719Z/datasets/custom_npz_builder_n256/datasets/N256/gs1/train.npz`
- Test NPZ: `outputs/hybrid_resnet_structural_rerun_20260226T110719Z/datasets/custom_npz_builder_n256/datasets/N256/gs1/test.npz`

This pair is the same `custom_npz_pair_n256` dataset referenced by the hybrid-resnet mode/skip/structural-search docs.

## Provenance

- The search design defines `custom_npz_pair_n256` as the caller-supplied `N=256` NPZ pair used in the architecture-search workflow.
- The runbook resolves that profile directly to `--train-npz` and `--test-npz`.
- The current archived pair above was generated by `grid_lines_workflow` and carries metadata showing:
  - `N=256`
  - `gridsize=1`
  - `nphotons=1e9`
  - `nimgs_train=2`
  - `nimgs_test=1`
  - `probe_source=custom`
  - `coords_type=relative`

## How To Use It

Preferred path for a single-model experiment:

```bash
python scripts/studies/grid_lines_torch_runner.py \
  --train-npz outputs/hybrid_resnet_structural_rerun_20260226T110719Z/datasets/custom_npz_builder_n256/datasets/N256/gs1/train.npz \
  --test-npz outputs/hybrid_resnet_structural_rerun_20260226T110719Z/datasets/custom_npz_builder_n256/datasets/N256/gs1/test.npz \
  --output-dir <output_dir> \
  --architecture hybrid_resnet \
  --seed 3 \
  --epochs 10 \
  --N 256 \
  --gridsize 1 \
  --fno-modes 12 \
  --fno-width 32 \
  --fno-blocks 4 \
  --no-hybrid-skip-connections \
  --hybrid-downsample-steps 2 \
  --hybrid-downsample-op stride_conv \
  --hybrid-resnet-blocks 6 \
  --hybrid-skip-style add \
  --no-probe-mask \
  --no-torch-mae-pred-l2-match-target
```

Runbook path when you need the staged sweep/orchestration machinery:

```bash
python scripts/studies/runbooks/run_hybrid_resnet_mode_skip_sweep.py \
  --ns 256 \
  --dataset-profiles-n256 custom_npz_pair_n256 \
  --custom-n256-train-npz outputs/hybrid_resnet_structural_rerun_20260226T110719Z/datasets/custom_npz_builder_n256/datasets/N256/gs1/train.npz \
  --custom-n256-test-npz outputs/hybrid_resnet_structural_rerun_20260226T110719Z/datasets/custom_npz_builder_n256/datasets/N256/gs1/test.npz \
  --allow-n256-direct-diagnostic \
  --top-k-n256 0 \
  ...
```

Important:

- For canonical non-diagnostic `N=256` promotion runs, the runbook requires both `cameraman256_halfsplit_v1` and `custom_npz_pair_n256`.
- For a lines-only architecture experiment, use the direct Torch runner or an explicitly diagnostic runbook invocation.
- Keep `--no-probe-mask` unless you are intentionally studying probe-mask effects.

## Metric Guidance

For the simplified architecture-improvement agent prompt, the optimization metric is `amp_ssim`.

- Treat `amp_ssim` as the only keep/discard metric.
- Supporting metrics may be logged, but they do not override a higher valid `amp_ssim`.
